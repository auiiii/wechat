# 动友 SportPal - 服务端接口文档 v2

## 一、概述

本文档定义了「动友 SportPal」微信小程序前端所需的全部服务端接口。

- **服务端地址**: `https://onejob.site`（ICP备案完成后启用）
- **开发环境**: `http://180.76.119.15:<端口>`
- **协议**: HTTPS（生产环境必须）
- **数据格式**: JSON
- **字符编码**: UTF-8

> **v2 变更说明**: 首页从「需求列表」改为「朋友圈动态流」，去掉联系发布者、交易类功能，新增评论、点赞、图片上传能力。打卡新增照片上传。

## 二、鉴权机制

### 2.1 微信登录鉴权流程

```
小程序                     你的服务端                   微信服务器
  │  wx.login()               │                           │
  │ ───────────────────────────────────────────────────────>│
  │ <── 返回 code ───────────────────────────────────────── │
  │                           │                           │
  │ POST /api/auth/wx-login {code}                         │
  │ ─────────────────────────>│                           │
  │                           │  jscode2session            │
  │                           │ ─────────────────────────>│
  │                           │ <── openid + session_key ──│
  │                           │ 生成JWT token              │
  │ <── {token, userInfo} ───│                           │
```

### 2.2 Token 使用方式

```
Authorization: Bearer <token>
```

### 2.3 服务端微信侧对接

```
GET https://api.weixin.qq.com/sns/jscode2session
  ?appid=wxc3a0e0b55a494e89
  &secret=<你的appSecret>
  &js_code=<前端传来的code>
  &grant_type=authorization_code
```

> ⚠️ `appSecret` 和 `session_key` 绝不能传到前端！

## 三、通用响应格式

### 成功
```json
{ "code": 0, "message": "success", "data": { ... } }
```

### 失败
```json
{ "code": 40001, "message": "错误描述", "data": null }
```

### 通用错误码

| 错误码 | 说明 |
|-------|------|
| 0 | 成功 |
| 40001 | 参数错误 |
| 40100 | 未登录 / Token 无效 |
| 40101 | Token 已过期 |
| 50000 | 服务器内部错误 |

## 四、接口详情

### 4.1 鉴权模块

#### POST `/api/auth/login` - 账号密码登录

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**返回：**
```json
{
  "code": 0,
  "data": {
    "token": "jwt_token_string",
    "userInfo": {
      "id": "user_001",
      "nickname": "动友用户",
      "avatar": "头像URL",
      "phone": "138****0000"
    }
  }
}
```

---

#### POST `/api/auth/register` - 账号密码注册

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，4-32位 |
| password | string | 是 | 密码，6-32位，bcrypt 加密存储 |
| phone | string | 是 | 手机号（11位），唯一 |

**服务端要求：** 用户名与手机号均唯一；注册成功即返回登录态（token），无需再调登录接口；注册后即可使用手机号验证码登录。

**返回：** 同登录返回，`isNewUser` 为 true。

```json
{
  "code": 0,
  "data": {
    "token": "jwt_token_string",
    "userInfo": {
      "id": "user_001",
      "nickname": "动友用户",
      "avatar": "",
      "phone": "138****0000"
    },
    "isNewUser": true
  }
}
```

**失败示例：**
```json
{ "code": 1, "message": "用户名已存在", "data": null }
{ "code": 1, "message": "手机号已注册", "data": null }
{ "code": 1, "message": "手机号格式不正确", "data": null }
```

---

#### POST `/api/auth/phone-login` - 手机号验证码登录

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| phone | string | 是 | 手机号（11位） |
| code | string | 是 | 短信验证码 |

**返回：** 同上

---

#### POST `/api/auth/sms-code` - 发送短信验证码

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| phone | string | 是 | 手机号 |

**服务端要求：** 验证码缓存5分钟，同号60秒内不可重发。

---

#### POST `/api/auth/wx-login` - 微信登录

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | string | 是 | wx.login() 返回的 code |

**服务端流程：** 调用 jscode2session → 获取 openid → 查询/注册用户 → 生成 JWT。

### 4.2 图片上传模块

#### POST `/api/upload/image` - 上传图片

**需要鉴权**: 是

**请求格式**: `multipart/form-data`，字段名 `file`

**返回：**
```json
{
  "code": 0,
  "data": {
    "url": "https://onejob.site/uploads/xxx.jpg"
  }
}
```

**服务端要求：**
- 支持 JPG/PNG/WEBP
- 单文件最大 5MB
- 暂时存储本地，docker启动挂载，后续可以使用minio

### 4.3 动态流模块

#### POST `/api/feed/create` - 发布动态

**需要鉴权**: 是

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| exerciseType | string | 是 | 运动类型: running/cycling/gym/yoga/swimming/other |
| content | string | 是 | 动态内容，最大500字 |
| images | string[] | 否 | 图片URL数组，最多9张（先调上传接口获取URL） |
| locationName | string | 否 | 地点名称 |

```json
  "data": {
    "id": "feed_001",
    "exerciseType": "running",
    "content": "今天晨跑5公里...",
    "images": ["https://..."],
    "locationName": "朝阳公园",
    "likes": 0,
    "commentCount": 0,
    "createdAt": "2025-06-08T10:00:00Z"
  }
}
```

---

## 五、数据库表设计

### users 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| open_id | VARCHAR(64) | 微信 OpenID，唯一索引，可空（账号密码/手机号注册用户为空） |
| username | VARCHAR(64) | 用户名，唯一索引，可空（账号密码登录用） |
| nickname | VARCHAR(50) | 昵称 |
| avatar | VARCHAR(500) | 头像URL |
| phone | VARCHAR(20) | 手机号 |
| password_hash | VARCHAR(200) | 密码哈希（bcrypt） |
| created_at | DATETIME | 创建时间 |

### feeds 动态表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 |
| user_id | VARCHAR(36) | 外键 → users.id |
| exercise_type | VARCHAR(20) | 运动类型 |
| content | TEXT | 动态内容 |
| images | JSON | 图片URL数组 |
| location_name | VARCHAR(100) | 地点名称 |
| likes | INT | 点赞数 |
| comment_count | INT | 评论数 |
| view_count | INT | 浏览数 |
| created_at | DATETIME | 创建时间 |

### feed_comments 评论表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 |
| feed_id | VARCHAR(36) | 外键 → feeds.id |
| user_id | VARCHAR(36) | 外键 → users.id |
| content | VARCHAR(200) | 评论内容 |
| likes | INT | 点赞数 |
| created_at | DATETIME | 创建时间 |

### feed_likes 点赞表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 |
| feed_id | VARCHAR(36) | 动态ID |
| user_id | VARCHAR(36) | 用户ID |
| comment_id | VARCHAR(36) | 评论ID（NULL表示点赞动态） |
| created_at | DATETIME | 创建时间 |

**唯一约束**: `(user_id, feed_id)` 或 `(user_id, comment_id)` 防止重复点赞。

### checkins 打卡记录表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 |
| user_id | VARCHAR(36) | 外键 → users.id |
| exercise_type | VARCHAR(20) | 运动类型 |
| images | JSON | 照片URL数组 |
| location_name | VARCHAR(100) | 地点名称 |
| note | VARCHAR(200) | 备注 |
| created_at | DATETIME | 打卡时间 |

## 六、JWT Token 规范

- **过期时间**: 7天
- **签名算法**: HS256
- **Payload**: `{ userId, openId, iat, exp }`
