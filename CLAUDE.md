# 项目说明
这是一个微服务后端java项目，使用springBoot:2.7.1和springCloud：2021.0.3版本构建。

# 代码规范
- 避免使用System.out.print打印日志
- 不接受: "先这样绕过"、"加个判断跳过"等绕过根因的解决问题方案
- 数据库字段设计考虑合理，参照数据库设计三大范式
- 避免使用大事务、长事务、慢sql、数据查询不分批等造成数据库负担的操作，通常情况不需要考虑分库分表
- 根据业务合理使用缓存，需要注意数据库&缓存一致性问题，通常我会使用过期消失，你可以结合业务场景自行选择方案
- 需考虑线程安全问题，合理使用线程池提升效率，同时建议自定义线程池，避免使用Executors工程类（内存泄露问题）
- 如果考虑引入消息队列进行削峰处理，请先同我确认后再进行代码改动并给出理由
- 代码应该是优美的、可读的、令人愉悦的

# 技术选型原则
- 如果产生依赖版本冲突，需要升级springBoot大版本或者springCloud大版本，请提示我确认后更改
- 涉及引入中间件时需考虑是否是成熟方案，社区活跃、文档完善吗？
- 中间件版本除了考虑版本兼容外，需要考虑是否有安全漏洞风险，比如log4j
